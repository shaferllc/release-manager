Documentation

Overview

Release Manager helps you version, tag, and release multiple projects from one place. It supports npm, Rust (Cargo.toml), Go (go.mod), Python (pyproject.toml / setup.py), and PHP (composer.json). Add project folders, see version and git status, then bump and push or tag-and-push.

Adding projects

Click Add project in the nav and choose a folder that contains one of:

  • package.json (npm/Node)
  • Cargo.toml (Rust)
  • go.mod (Go)
  • pyproject.toml or setup.py (Python)
  • composer.json (PHP)

The app detects the type and shows name and version. Projects are stored in app data; use Remove on a project to drop it from the list.

Project detail

Select a project to see its detail. Sections are collapsible (click the header to expand/collapse).

  • Git — Branch, ahead/behind, uncommitted files. If there are changes, you can enter a commit message and Commit from the app.
  • Version & release — Current version and latest tag; list of released versions (with links and Download per version for GitHub repos). Release creates a git tag and pushes it; pushing the tag triggers GitHub Actions to build and attach the DMG (and other installers) to the release. For npm: Patch / Minor / Major / Pre-release bump then tag and push. For Rust, Go, Python, PHP: Tag and push uses the current version in the manifest (no bump; update the version in the file first if needed). Release notes, draft, and pre-release options use your GitHub token when set. The app shows recent commits and an optional suggested bump from conventional commits (feat: → minor, fix: → patch, breaking change → major); you choose the bump.
  • Sync & download — Sync runs git fetch. Download latest / Choose version… fetch GitHub Release assets for your platform.

Header actions:

  • Open in Terminal — Opens the system terminal with the project folder as the working directory.
  • Open in editor — Opens the project in Cursor or VS Code (whichever is on your PATH).
  • Open in Finder — Reveals the project folder in the file manager.
  • Copy path — Copies the project path to the clipboard. Remove — Removes the project from the list.

Keyboard shortcuts (when a project is selected): ⌘1 Patch, ⌘2 Minor, ⌘3 Major, ⌘S Sync, ⌘D Download latest (Windows/Linux: use Ctrl instead of ⌘).

Dashboard

Click Dashboard in the sidebar to see a table of all projects: name, version, latest tag, branch, ahead/behind. Use Filter (All / Needs release) and Sort (Name / Needs release first). Click a row to open that project’s detail.

Batch release

Select two or more projects with the checkboxes in the sidebar. A batch bar appears with Patch, Minor, and Major. Click one to run that bump and release for all selected projects in sequence (with confirmation). Only npm projects are bumped; non-npm projects would need to be released individually with Tag and push.

Settings

Click Settings to set a GitHub token (optional) and Ollama (optional). The token gives higher API limits, access to private repos, and the ability to create or update GitHub releases (and drafts) from the app. See Generate with Ollama below for using a local model to generate commit messages and release notes.

Generate with Ollama

You can use a local Ollama model to generate commit messages and release notes.

1. Install and start Ollama

  • Install from ollama.com.
  • In a terminal, start the server: ollama serve (often runs in the background after install). If you see address already in use, Ollama is already running—use Generate without starting it again.
  • Pull a model, e.g. ollama pull llama3.2.

2. Set base URL and model in Settings

In Settings, set Ollama Base URL to http://localhost:11434 and Model to the name you pulled (e.g. llama3.2).

3. Generate in the project view

With a project selected, use Generate next to the commit message (Git section) to get a suggested conventional commit line, or Generate with Ollama next to release notes to get suggested notes from recent commits. Edit the result if needed.

Package managers

  • npm — Version from package.json. Release = bump (patch/minor/major) then commit, tag, push.
  • Rust — Version and name from Cargo.toml. Release = tag and push current version (update version in Cargo.toml manually first if needed).
  • Go — Name from go.mod module path; no version in go.mod. Use Tag and push only if you’ve set a version elsewhere; otherwise create tags manually.
  • Python — Version and name from pyproject.toml or setup.py. Release = tag and push current version.
  • PHP — Version and name from composer.json. Release = tag and push current version (set version in composer.json, e.g. "1.2.3", then use Tag and push).

Theme

Use the Dark / Light toggle in the nav bar. The choice is saved and applied on next launch.
